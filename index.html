Tier 1 — The Brainstem

Local agent server — the survival basics
Start here ▶

The biological brainstem keeps you alive — breathing, heartbeat, reflexes. Your AI brainstem does the same: it runs the core agent loop, calls tools, and responds. Everything else builds on top of this.

1. Authenticate with GitHub

Your GitHub Copilot subscription IS the AI engine. No OpenAI keys needed.

gh auth login

2. Start the brainstem

The server starts on port 7071 with a chat UI, health endpoint, and agent auto-discovery.

brainstem

3. Talk to it

Open localhost:7071 in your browser, or hit the API directly.

curl -s localhost:7071/health | python3 -m json.tool

4. Make it yours

Write a soul.md to define its personality. Drop *_agent.py files into the agents folder to give it skills. Point SOUL_PATH and AGENTS_PATH in .env to your private files.

Tier 2 — The Hippocampus

Azure Functions + persistent memory
Cloud-ready ▶

The hippocampus is where memories form. This tier gives your agent persistent memory, an agent library, and the Azure Functions runtime. It runs locally first — deploy to Azure when you're ready.

1. Create a project

One command clones CommunityRAPP, sets up a venv, installs dependencies, and creates a start script. No Azure account or API keys needed.

curl -fsSL https://raw.githubusercontent.com/kody-w/rapp-installer/main/community_rapp/install.sh | bash

Windows: irm https://raw.githubusercontent.com/kody-w/rapp-installer/main/community_rapp/install.ps1 | iex

2. Start it

The chat UI walks you through GitHub device-code auth. No manual env vars, no config files to edit.

cd ~/rapp-projects/my-project && ./start.sh

3. Talk to it

Open index.html in your browser. Send a message. Memories persist across conversations and restarts.

4. Deploy to Azure (when ready)

Edit local.settings.json with your Azure OpenAI credentials, then publish. Or use the ARM template for a full deployment.

func azure functionapp publish YOUR_APP_NAME --build remote

What you learn: Azure Functions, persistent memory, agent discovery, Azure OpenAI, managed identity.

Tier 3 — The Nervous System

Connect to Copilot Studio — reach into M365
Enterprise ▶

Your nervous system extends your brain's reach to every part of your body. Copilot Studio does the same: it connects your agent to Teams, M365 Copilot, and the entire Microsoft ecosystem.

1. Download the solution

Grab the Power Platform solution package. This .zip contains the declarative agent, connectors, and flows preconfigured for your Azure Function.

curl -fsSL -o MSFTAIBASMultiAgentCopilot.zip https://raw.githubusercontent.com/kody-w/rapp-installer/main/MSFTAIBASMultiAgentCopilot_1_0_0_5.zip

2. Import into Power Platform

Go to make.powerapps.com → Solutions → Import → upload the .zip.

3. Wire the connector

Open the Power Automate flow, update the HTTP URL to your Azure Function endpoint. Add your Function Key if using key-based auth.

4. Publish to Teams & M365 Copilot

In Copilot Studio, publish the agent. Same logic you tested locally — now enterprise-wide.

What you learn: Copilot Studio, declarative agents, Power Platform solutions, Teams integration, enterprise AI deployment.
