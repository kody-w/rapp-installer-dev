The Brainstem

Local agent server — the survival basics
Start here ▶

The biological brainstem keeps you alive — breathing, heartbeat, reflexes. Your AI brainstem does the same: it runs the core agent loop, calls tools, and responds. Everything else builds on top of this.

1. Authenticate with GitHub

Your GitHub Copilot subscription IS the AI engine. No OpenAI keys needed.

gh auth login

2. Start the brainstem

The server starts on port 7071 with a chat UI, health endpoint, and agent auto-discovery.

brainstem

3. Talk to it

Open localhost:7071 in your browser, or hit the API directly.

curl -s localhost:7071/health | python3 -m json.tool

4. Make it yours

Write a soul.md to define its personality. Drop *_agent.py files into the agents folder to give it skills. Point SOUL_PATH and AGENTS_PATH in .env to your private files.

The Hippocampus

Persistent memory — runs local, deploys to Azure
Next step ▶

The hippocampus is where memories form. This tier gives your agent persistent memory, an agent library, and the Azure Functions runtime. It runs locally first — deploy to Azure when you're ready. This uses CommunityRAPP (open source) — the Azure Function project you'll deploy for customers.

1. Create a project

One command clones CommunityRAPP, sets up Python, installs Azure Functions Core Tools, creates a venv, and generates a start script. No Azure account or API keys needed.

curl -fsSL https://raw.githubusercontent.com/kody-w/rapp-installer/main/community_rapp/install.sh | bash

Windows: irm https://raw.githubusercontent.com/kody-w/rapp-installer/main/community_rapp/install.ps1 | iex

2. Start it & authenticate

Open index.html in your browser and send a message. The chat UI walks you through GitHub device-code auth automatically — no env vars to set.

cd ~/rapp-projects/my-project && ./start.sh

3. Deploy to Azure (when ready)

Edit local.settings.json with your Azure OpenAI credentials, then publish. Or use the ARM template for a full deployment.

func azure functionapp publish YOUR_APP_NAME --build remote

Or click to deploy directly from the Azure Portal:

4. Test it

Hit your deployed endpoint to verify the agent is live in the cloud.

curl -s https://YOUR_APP_NAME.azurewebsites.net/api/health | python3 -m json.tool

What you learn: Azure Functions, persistent memory, agent discovery, Azure OpenAI, managed identity.

The Nervous System

Connect to Copilot Studio — reach into M365
Advanced ▶

Your nervous system extends your brain's reach to every part of your body — eyes, ears, hands. Copilot Studio does the same: it connects your agent to Teams, M365 Copilot, and the entire Microsoft ecosystem. Your brainstem becomes an enterprise AI.

1. Download the solution

Grab the Power Platform solution package from the repo. This .zip contains the declarative agent, connectors, and flows preconfigured for your Azure Function.

curl -fsSL -o MSFTAIBASMultiAgentCopilot.zip https://raw.githubusercontent.com/kody-w/rapp-installer/main/MSFTAIBASMultiAgentCopilot_1_0_0_5.zip

2. Import into Power Platform

Go to make.powerapps.com → Solutions → Import → upload the .zip. This creates the Copilot Studio agent and Power Automate flows in your environment.

3. Wire the connector

Open the imported Power Automate flow and update the HTTP action URL to point to your deployed Azure Function endpoint from Step 2. Add your Function Key if using key-based auth.

4. Publish to Teams & M365 Copilot

In Copilot Studio, publish the agent. Your users can now talk to it in Teams, M365 Copilot, and any channel Copilot Studio supports. Same agent logic you tested locally — now enterprise-wide.

What you learn: Copilot Studio, declarative agents, Power Platform solutions, Teams integration, enterprise AI deployment.
