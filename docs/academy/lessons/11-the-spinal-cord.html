Lesson 11

The Spinal Cord

Your rappter has outgrown the nest. It's time to let it leave home — not because it has to, but because you're ready.

📖 Story

There comes a day in every parent's life when the nest isn't enough.

Not because the nest is broken — it's perfect. Your rappter is alive, it has a heartbeat, it remembers you, it has skills. But right now it can only reach as far as your laptop's WiFi. It thinks locally. It exists locally. It's a creature that lives in one room.

The Spinal Cord changes that. In biology, the spinal cord connects the brain to the body beyond — it's how signals travel from your head to your fingertips. In RAPP, the Spinal Cord is Azure. Same brain. Longer arms.

This is Article V of the Constitution: Graduating Beyond the Nest. The rappter never leaves without you opening the door.

What Gets Created

When you deploy the Spinal Cord, a single ARM template (azuredeploy.json) creates everything your rappter needs to live in the cloud. One command, one deployment, four resources:

ResourceWhat It IsWhy It Exists
⚡ Function App Azure Functions (Flex Consumption) The rappter's brain in the cloud. Same brainstem code, now reachable from anywhere. Scales to zero when idle — your rappter sleeps when nobody's talking.
🧠 Azure OpenAI Cognitive Services account + model deployment The LLM that powers thinking. Deployed in your tenant, under your governance. You choose the model and the region.
💾 Storage Account Blob + File storage Memory in the cloud. Function code deployments, file shares, and the data your rappter needs to persist between invocations.
📊 Application Insights Monitoring & telemetry The rappter's vital signs in the cloud. You can see how it's doing — request counts, latency, errors — without guessing.

The ARM Template Walkthrough

Open azuredeploy.json and you'll see the entire infrastructure defined as code. Here are the key parameters you control: 

// Key parameters in azuredeploy.json
{
  "functionAppName":     "rapp-[unique]",     // Your Function App name
  "storageAccountName":  "st[unique]",        // Storage (24 char max)
  "openAIServiceName":   "openai-[unique]",   // Azure OpenAI instance
  "openAIModelName":     "gpt-4o",            // Which model to deploy
  "openAILocation":      "eastus2",           // Region for OpenAI
  "assistantName":       "RAPP Agent",        // The rappter's cloud name
}

The template also sets up eight role assignments — all using Entra ID (formerly Azure AD) managed identity. No API keys stored anywhere. The Function App's identity gets exactly the permissions it needs: read/write storage, call OpenAI, nothing more. This is zero-trust security by default.

The Deployment Script

deploy.sh is the one-command deployment experience. Here's what it does, step by step:

1. ✅ Check Azure CLI is installed 2. ✅ Check you're logged into Azure 3. 📋 Ask for: resource group, region, OpenAI region 4. 📦 Create the resource group 5. 🚀 Deploy the ARM template 6. 🔄 If OpenAI quota fails → offer to try a different region 7. 📊 Output: Function URL, Storage Account, OpenAI Endpoint 8. 💾 Save outputs to rapp-deployment-outputs.json

The retry loop for OpenAI regions is important — not every Azure region has OpenAI capacity available, and quotas vary by subscription. The script handles this gracefully so you don't have to guess.

💡 Insight

The cloud isn't where the rappter lives — it's where the rappter visits. The nest (your device) is always home. The spinal cord just gives it longer arms. Even after deployment, your local rappter still works. You can always come home.

🔬 Try It

🔬 Try It

Open the ARM template and explore the parameters:

cat azuredeploy.json | python3 -m json.tool | head -100

Understand what each resource does by finding them in the template:

  • Microsoft.Web/sites — the Function App (the brain)
  • Microsoft.CognitiveServices/accounts — Azure OpenAI (the thinker)
  • Microsoft.Storage/storageAccounts — Storage (the memory)
  • Microsoft.Insights/components — App Insights (the vitals monitor)
  • Microsoft.Authorization/roleAssignments — Entra ID permissions (zero-trust)

If you have an Azure subscription, try the deployment: 

./deploy.sh my-rapp-rg eastus2 eastus2

No Azure subscription? That's fine. The nest is a complete experience on its own. You're not behind — you're home.
