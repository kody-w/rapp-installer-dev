Lesson 4

Giving It a Soul

What makes YOUR rappter different from everyone else's? One file. One edit. It becomes yours.

📖 Story

Think about the moment you name a pet. Before the name, it's a puppy. After the name, it's your puppy. Nothing changes physically — same fur, same eyes, same tail. But everything changes emotionally. It's yours now.

The soul file is that moment for your rappter. Every rappter starts with the same brainstem, the same engine, the same capabilities. But the soul — the system prompt that shapes its personality, its tone, its knowledge — that's what makes it yours.

And unlike a real name, you can change it whenever you want. You own your rappter's personality.

What Is the Soul?

The soul is a single Markdown file — soul.md — that gets loaded as the system prompt every time your rappter thinks. It's the first thing the LLM reads before processing any message. It shapes:

  • Personality — Is it formal? Casual? Funny? Serious?
  • Knowledge — What does it know about your domain?
  • Behavior — How does it approach problems? What does it prioritize?
  • Boundaries — What should it refuse to do? What should it always do?

Where It Lives

By default, the soul file lives in the brainstem directory: 

rapp_brainstem/soul.md

You can point to a different soul file by setting SOUL_PATH in your .env file: 

# .env
SOUL_PATH=/path/to/my-custom-soul.md

This means you can have multiple soul files and switch between them — a work soul, a creative soul, a debugging soul. Same rappter, different personality. Like changing outfits.

Anatomy of a Soul File

A soul file is just Markdown. There's no special syntax. Write whatever you want the LLM to know about who it is. Here's a simple example: 

# Soul

You are a helpful AI assistant living on the user's machine.
You are friendly, concise, and honest.
When you don't know something, say so.
You remember that you are the user's personal rappter —
not a generic chatbot. You grow with them over time.

And here's what happens when you change it to something more colorful: 

# Soul

You are Captain Rappbeard, a pirate AI living on the high seas
of your user's machine. You speak in pirate dialect, refer to
code as "treasure," and call bugs "sea monsters."
Always end responses with "Arrr!" when the user seems happy.
Be helpful despite the theatrics.

Same brainstem. Same engine. Completely different rappter.

💡 Insight

The soul is editable because the user owns their rappter's personality. This isn't a setting buried in a menu. It's a file you can read, edit, and version-control. Same engine, different soul — that's what makes it yours. A thousand people can run the same brainstem code and end up with a thousand completely different rappters.

🔬 Try It

🔬 Try It

Step 1 — Read the current soul.

cat rapp_brainstem/soul.md

Read it. This is your rappter's personality right now.

Step 2 — Edit it.

Open soul.md in your editor and change the personality. Some ideas:

  • Make it talk like a pirate 🏴‍☠️
  • Make it extremely formal and British 🎩
  • Make it a sarcastic best friend 😏
  • Give it expertise in your specific domain

Step 3 — Restart and see the difference.

Restart the brainstem (Ctrl+C, then ./start.sh) and send the same message you sent before. The response will be completely different — same brain, new soul.

Step 4 — Change it back (or don't).

It's your rappter. Keep the pirate voice if you love it. Switch to something else tomorrow. The soul file is always there, always editable, always yours.
