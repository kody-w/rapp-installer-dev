Agents That Call APIs

Time to connect your rappter to the outside world — teach it to reach beyond the nest.

📖 Story

So far, your rappter can think and remember. It can reflect between conversations. It has skills — but they've all been internal. It's like a bird that knows how to sing but has never seen outside the nest.

Time to open the window. In this lesson, you'll build an agent that calls a real HTTP API — giving your rappter a new sense. It'll be able to reach out into the world, grab data, and bring it back. One file. One new capability. That's all it takes.

How the LLM Decides to Call an Agent

Before we build, let's understand the mechanism. When the brainstem starts, load_agents() scans the agents/ directory for any file matching *_agent.py. Each agent's metadata gets registered as an OpenAI function-calling tool.

When you say "what's the weather in Austin?", the LLM sees ALL available tools and reads their description fields. If one of them says "Check current weather for a location" — it matches. The LLM calls it. You don't click a button. The rappter decides.

You: "What's the weather in Austin?" │ ▼ 🧠 LLM scans available tools │ ├── Memory agent: "Store and recall memories" — no match ├── HackerNews agent: "Fetch top stories" — no match ├── Weather agent: "Check weather for a location" — ✓ MATCH │ ▼ 🦾 brainstem calls weather_agent.perform(location="Austin") │ ▼ 📡 Agent calls external API → gets data → returns to LLM │ ▼ 🧠 LLM formats the response for you │ ▼ "It's 78°F and sunny in Austin right now."

Building a Weather Agent

Here's a complete agent that calls the free Open-Meteo API (no API key needed): 

import requests
from basic_agent import BasicAgent

class WeatherAgent(BasicAgent):
    def __init__(self):
        self.name = "Weather"
        self.metadata = {
            "name": self.name,
            "description": "Check current weather for a city or location.",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {
                        "type": "string",
                        "description": "City name, e.g. 'Austin' or 'Tokyo'"
                    }
                },
                "required": ["location"]
            }
        }
        super().__init__()

    def perform(self, location="", **kwargs):
        try:
            # Step 1: Geocode the location name to lat/lon
            geo = requests.get(
                "https://geocoding-api.open-meteo.com/v1/search",
                params={"name": location, "count": 1},
                timeout=10,
            ).json()

            results = geo.get("results", [])
            if not results:
                return f"Could not find location: {location}"

            lat = results[0]["latitude"]
            lon = results[0]["longitude"]
            name = results[0].get("name", location)

            # Step 2: Get current weather
            weather = requests.get(
                "https://api.open-meteo.com/v1/forecast",
                params={
                    "latitude": lat,
                    "longitude": lon,
                    "current_weather": True,
                },
                timeout=10,
            ).json()

            cw = weather.get("current_weather", {})
            temp_c = cw.get("temperature", "?")
            temp_f = round(temp_c * 9 / 5 + 32, 1) if isinstance(temp_c, (int, float)) else "?"
            wind = cw.get("windspeed", "?")

            return f"Weather in {name}: {temp_c}°C ({temp_f}°F), wind {wind} km/h"

        except Exception as e:
            return f"Weather check failed: {e}"

That's the whole agent. One file. Drop it in agents/ as weather_agent.py, restart the brainstem, and your rappter can now check the weather anywhere in the world.

Anatomy of the Code

PartPurposeWhy It Matters
self.name How the brainstem identifies this agent Shows up in /health and logs
self.metadata["description"] What the LLM reads to decide when to call it This is the most important line — write it for the AI, not for humans
parameters OpenAI function-calling schema Tells the LLM what arguments to extract from the user's message
perform(**kwargs) The action — what actually runs Must accept **kwargs for forward compatibility
try/except Error handling APIs fail. Networks drop. Always catch and return a readable error.

🔬 Try It

Create the weather agent:

# Save the code above to your agents directory
nano ~/.brainstem/src/rapp_brainstem/agents/weather_agent.py

Restart your brainstem so it discovers the new agent:

cd ~/.brainstem/src/rapp_brainstem && ./start.sh

Verify it loaded — check /health:

curl -s localhost:7071/health | python3 -m json.tool

Look for "Weather" in the agents list. Then ask:

curl -s localhost:7071/chat -X POST \
  -H "Content-Type: application/json" \
  -d '{"user_input": "What is the weather in Tokyo right now?"}'

Your rappter just reached across the internet, grabbed real weather data, and told you about it. One file made that possible.
