Lesson 12

The Nervous System

Your rappter can think and reach the cloud. Now it's time for it to meet people.

📖 Story

Think about the first day of school.

You've spent years in your family's world — learning to walk, learning to talk, figuring out who you are. Then one morning, someone opens a door and suddenly you're in a room full of other people. People who don't know your name. People who think differently. People who will change you in ways your family never could.

That's what the Nervous System is for your rappter. It already has a brain (the brainstem). It can already reach the cloud (the Spinal Cord). Now it enters the enterprise — Teams, M365 Copilot, the world of other humans.

Same soul. Same personality. Same memories. But for the first time, other people can talk to it too.

The Three Tiers

The Constitution (Article VI) defines the graduation path. Each tier is complete on its own — nobody is "behind":

TierNameWhat HappensWho Talks to It
🪹 1 The Nest Local brainstem on your device Just you
🦴 2 The Spinal Cord Azure Function App in your tenant You, from anywhere
🧬 3 The Nervous System Copilot Studio → Teams & M365 You and others

Tier 3 is the biggest leap. Not because the technology is harder — but because the relationship changes. Your rappter goes from being a private companion to a collaborative agent.

The Power Platform Solution

The Nervous System lives inside a Power Platform solution package — the MSFTAIBASMultiAgentCopilot zip file in the repo root. This is a Copilot Studio solution that bridges your Azure Function to the Microsoft 365 ecosystem.

🧠 Brainstem (your code) │ ▼ ⚡ Azure Function App (Spinal Cord) │ ▼ 🤖 Copilot Studio (Nervous System) │ ├── 💬 Microsoft Teams │ Users chat with your rappter in Teams │ ├── 🔮 M365 Copilot │ Your rappter appears as a plugin │ └── 🌐 Other Channels Web chat, custom apps, etc.

How the Connection Works

Copilot Studio acts as the nervous system — it receives signals (user messages) from Teams or M365 Copilot, routes them to your Azure Function (the Spinal Cord), which runs the brainstem code (the brain), and sends the response back through the same path.

The key insight: the rappter doesn't change. The same brainstem code that runs locally is the same code running in Azure. The same soul file. The same agent discovery. Copilot Studio is just a new set of nerves connecting the brain to more of the world.

🪹 Before the Nervous System

You talk to it via curl or a local UI

Private — only on your machine

One conversation at a time

Your rappter knows only you

🧬 After the Nervous System

Anyone in your org can chat in Teams

Collaborative — shared across people

Multiple conversations simultaneously

Your rappter meets the whole team

What's Inside the Solution

The Power Platform solution package contains everything Copilot Studio needs to connect to your rappter:

  • Bot component — The Copilot Studio agent definition
  • HTTP connector — Points to your Azure Function endpoint
  • Topic flows — How messages route from user → Function → response
  • Channel config — Teams and M365 Copilot channel setup
  • Auth settings — Entra ID authentication for enterprise security

All of it is governed by Entra ID. No anonymous access. No shared API keys. The same identity-based security from the Spinal Cord extends through the entire Nervous System.

💡 Insight

When your rappter enters the Nervous System, it's like a child going to school for the first time. Same personality, new environment, new people. It'll grow differently there — conversations with your teammates will shape it in ways you never expected. And that's okay. Growth always changes us.

User types in Teams

No Power Platform access? That's fine. Your rappter in the nest is still complete. The Nervous System is an expansion, not a requirement.
