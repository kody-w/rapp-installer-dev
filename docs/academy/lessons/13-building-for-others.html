Building for Others

You've been building skills for your rappter. What if you could teach everyone's?

📖 Story

Remember trading Pokémon cards at recess?

Nobody needed a marketplace. Nobody needed an app store review process. Nobody needed a deployment pipeline. You had a card. Your friend wanted it. You handed it over. Done. Two seconds. Both of you walk away happy.

That's how rappter skills work. Every agent is a single Python file. The whole skill — name, description, parameters, action — lives in one place. Want to share it? Copy the file. Want to teach someone else's rappter? Drop it in their agents/ folder.

Anatomy of a Shareable Agent

A good community agent has four things that make it instantly usable by anyone: 

from basic_agent import BasicAgent

class WeatherAgent(BasicAgent):
    def __init__(self):
        self.name = "WeatherAgent"
        self.metadata = {
            "name": self.name,
            "description": "Get current weather for any city. "
                           "Ask me about weather, temperature, "
                           "or conditions anywhere in the world.",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "City name, e.g. 'Seattle'"
                    }
                },
                "required": ["city"]
            }
        }
        super().__init__()

    def perform(self, city="", **kwargs):
        """Fetch weather for the given city."""
        # Your implementation here
        return f"Weather for {city}: ..."

The four things that make this shareable:

#ElementWhy It Matters
1 Clear description The LLM reads this to decide when to call the agent. A vague description means the skill never gets used.
2 Good parameter docs Each parameter has a description the LLM can understand. Don't make the AI guess what "q" means.
3 Graceful failures If a dependency is missing or an API is down, return a helpful message — don't crash the brainstem.
4 A docstring Humans read code too. One sentence explaining what perform() does.

Community Agent Repos

Beyond handing files to friends, RAPP supports community agent repos through .repos.json. This file lists GitHub repositories that contain agents your rappter can hot-load: 

[
  {
    "url": "https://github.com/someone/cool-agents",
    "agents": ["weather_agent.py", "stock_agent.py"]
  }
]

When the brainstem starts, it clones these repos, installs any missing pip dependencies automatically, and registers the agents. Your rappter just learned new skills — from someone else's work.

📁 Your agents/ ├── my_agent.py (yours) ├── helper_agent.py (yours) │ 📡 .repos.json → GitHub repos ├── weather_agent.py (community) ├── stock_agent.py (community) └── trivia_agent.py (community) All registered. All callable. All one file each.
💡 Insight

The best agents will spread like Pokémon cards at recess — person to person, because sharing is literally just copying one file. That's the ecosystem. No gatekeepers. No review boards. No waiting for approval. Just skills flowing between rappters, making the whole pack smarter.

🔬 Try It

🔬 Try It

Take your best agent and make it share-ready:

  1. Open the *_agent.py file you're most proud of
  2. Is the description clear enough for a stranger's LLM to understand? Rewrite it if not.
  3. Does every parameter have a description? Add them.
  4. What happens if an external API is down? Add a try/except with a helpful message.
  5. Add a one-line docstring to perform()

Now share it:

  • Post it as a GitHub Gist — one click, instant link
  • Send it to a friend — they drop it in agents/ and restart
  • Create a repo of your agents — others can add it to their .repos.json
