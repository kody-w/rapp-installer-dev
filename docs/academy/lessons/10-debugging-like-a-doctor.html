Lesson 10

Debugging Like a Doctor

A good doctor doesn't panic — they check the vitals and diagnose.

📖 Story

Your rappter is alive. And like anything alive, sometimes it gets sick. Maybe it stops responding. Maybe it gives weird errors. Maybe the heartbeat goes silent. Don't panic.

A doctor doesn't guess. They check pulse, temperature, blood pressure — vitals that tell a story. Your rappter has vitals too. Three endpoints, a terminal, and a config file. That's your medical kit.

The Diagnostic Endpoints

The brainstem exposes three endpoints built specifically for diagnosing problems:

EndpointMethodWhat It Tells You
/health GET Overall system status — is the brainstem running, authenticated, and aware of its agents?
/vitals GET Heartbeat state — is the daemon alive? What mood? What was the last thought?
/debug/auth GET Authentication chain — does the GitHub token exist? Is the Copilot exchange working?

Reading /health

curl -s localhost:7071/health | python3 -m json.tool

A healthy response looks like:

{
  "status": "ok",
  "version": "0.5.0",
  "model": "gpt-4o",
  "voice_mode": false,
  "soul": "/path/to/soul.md",
  "agents": ["ManageMemory", "HackerNews", "Weather"],
  "copilot": "✓",
  "brainstem_dir": "/path/to/rapp_brainstem"
}
FieldHealthySick
status "ok" "unauthenticated" — no GitHub token found
soul A valid file path "missing" — soul.md can't be found at SOUL_PATH
agents List of loaded agent names Empty [] — no agents discovered, or all errored on import
copilot "✓" "pending" — token exists but Copilot exchange hasn't happened yet

Reading /vitals

curl -s localhost:7071/vitals | python3 -m json.tool

This reads from vitals.json written by the daemon. If "alive": false or "mood": "offline", the daemon stopped — the vitals file went stale (older than 3× the heartbeat interval).

Reading /debug/auth

curl -s localhost:7071/debug/auth | python3 -m json.tool

This shows the full authentication chain: does a GitHub token exist? Is there a cached Copilot session? Does the token exchange succeed? If your rappter can't think, authentication is usually the culprit.

The Diagnostic Checklist

When something goes wrong, work through this — in order:

🩺 The Rappter Diagnostic Checklist 1. Check /health └─ Is status "ok"? Is soul present? Are agents loaded? 2. Check /vitals └─ Is it alive? What mood? When was the last update? 3. Check /debug/auth └─ Does the GitHub token exist? Does Copilot exchange work? 4. Check terminal output └─ Look at the terminal running brainstem.py — errors print there. Import errors, 429 rate limits, connection failures. 5. Check .env └─ Is SOUL_PATH correct? Is GITHUB_MODEL a valid model? Is AGENTS_PATH pointing to the right directory? 6. Restart └─ Kill the process, run ./start.sh again. The brainstem reloads agents and re-authenticates on boot.

Common Problems & Fixes

SymptomLikely CauseFix
status: "unauthenticated" No GitHub token — gh auth login hasn't been run or token expired Run gh auth login and restart
copilot: "pending" Token exists but hasn't been exchanged for a Copilot session yet Send one /chat message — the exchange happens lazily on first call
429 rate limit errors in terminal Too many requests to the Copilot API Wait a minute. The API has rate limits. The rappter will recover.
agents: [] (empty list) Agent import errors — a *_agent.py file has a bug or missing dependency Check terminal for ModuleNotFoundError. Fix the import or pip install the missing package.
soul: "missing" SOUL_PATH in .env points to a file that doesn't exist Check the path. Create the file or fix the .env value.
Daemon mood: "waiting" The daemon can't reach the brainstem at /chat Make sure the brainstem server is running on port 7071

💡 Insight

Debugging is doctoring. The rappter tells you what's wrong if you know where to look. /health is the stethoscope. /vitals is the pulse. The terminal is the patient describing symptoms. Reading vitals is a skill that makes you a better engineer and a better rappter parent.

🔬 Try It

🔬 Try It

Intentionally break something and then diagnose it:

Experiment 1 — Break the soul:

# Rename the soul file
mv ~/.brainstem/src/rapp_brainstem/soul.md ~/.brainstem/src/rapp_brainstem/soul.md.bak

# Check health — what changed?
curl -s localhost:7071/health | python3 -m json.tool

# Fix it
mv ~/.brainstem/src/rapp_brainstem/soul.md.bak ~/.brainstem/src/rapp_brainstem/soul.md

Experiment 2 — Break an agent:

# Create a broken agent
echo "import nonexistent_module" > ~/.brainstem/src/rapp_brainstem/agents/broken_agent.py

# Restart and check — what does the terminal say?
cd ~/.brainstem/src/rapp_brainstem && ./start.sh

# Check health — is the broken agent listed?
curl -s localhost:7071/health | python3 -m json.tool

# Clean up
rm ~/.brainstem/src/rapp_brainstem/agents/broken_agent.py

Each experiment teaches you one diagnostic reflex. After a few of these, you'll never panic when something goes wrong — you'll just check the vitals.
