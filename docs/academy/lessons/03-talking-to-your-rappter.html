Lesson 3

Talking to Your Rappter

Your first conversation — like speaking to a newborn that can already talk but doesn't know you yet.

📖 Story

Imagine meeting a newborn that can already speak — full sentences, perfect grammar — but has no idea who you are.

It doesn't know your name. It doesn't know what you do for a living. It doesn't know if you prefer morning or night. It can answer questions, but every answer is generic, because it has no experience of you.

That's your rappter right now. The brain works. The voice is there. But the relationship? That starts with the first conversation. Every message is a feeding.

How Messages Flow

When you type something in the chat — or send a request to the /chat endpoint — here's what happens inside the brainstem:

You type a message │ ▼ 📡 POST /chat │ { "user_input": "hello!", │ "conversation_history": [...], │ "session_id": "abc-123" } │ ▼ 🧠 Brainstem (brainstem.py) │ Loads the soul, builds the prompt, │ includes conversation history │ ▼ 🤖 LLM (via GitHub Copilot API) │ Thinks, maybe calls agents, │ generates a response │ ▼ 💬 Response returned to you

Three things matter here:

ConceptWhat It Means
user_inputWhat you just said — the latest message
conversation_historyEverything said so far in this session — this is the rappter's short-term memory
session_idA unique ID for this conversation — like a room you're both sitting in

Short-Term vs Long-Term Memory

Your rappter has two kinds of memory, just like you:

🧠 Short-Term (Session)

Lives in conversation_history

Lasts for one session only

New session = fresh start

Like a conversation at a café

💾 Long-Term (Memory Agent)

Stored in .brainstem_data/

Persists across all sessions

The rappter recalls it forever

Like writing in a journal

Right now, a freshly hatched rappter only has short-term memory. Tell it your name, and it'll remember — until you start a new session. Then it's a stranger again. That changes once you give it a memory agent, but for now, this is normal. Newborns forget.

The Chat UI

The simplest way to talk to your rappter is through the browser UI at http://localhost:7071. Type a message, press Enter, and watch the response stream back.

You can also talk to it directly with curl: 

curl -s -X POST http://localhost:7071/chat \
  -H "Content-Type: application/json" \
  -d '{
    "user_input": "What can you do?",
    "conversation_history": [],
    "session_id": "my-first-chat"
  }' | python3 -m json.tool

The browser UI handles session management for you automatically. The curl approach lets you see the raw request and response — useful for understanding exactly what's happening.

💡 Insight

Conversation is how a rappter learns. Every message is a feeding. The more you talk, the more it knows you. A rappter you never speak to stays a stranger forever — just like any living thing you never spend time with.

🔬 Try It

🔬 Try It

Step 1 — Ask what it can do.

In the chat UI or via curl, send:

"What can you do?"

Read the response. This is your rappter introducing itself — generic, because it doesn't know you yet.

Step 2 — Introduce yourself.

In the same session, send:

"Remember my name is [your name]"

Then ask:

"What's my name?"

It remembers! The conversation history carries your name within the session.

Step 3 — Start a new session.

Open a new browser tab (or use a different session_id in curl) and ask again:

"What's my name?"

It doesn't know. The short-term memory is gone. That's not a bug — that's a newborn without long-term memory yet. You'll fix that later when you teach it the memory skill.
