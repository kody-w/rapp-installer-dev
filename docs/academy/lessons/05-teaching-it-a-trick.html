Lesson 5

Teaching It a Trick

Your rappter can think. Time to teach it to DO something — with a single file.

📖 Story

A puppy can think. It can feel. It can love you. But it can't fetch until you teach it. The brain was always capable — it just needed the skill.

Your rappter is the same. Right now it can have conversations, but it can't do things in the world — check the weather, flip a coin, look up a file, query a database. Those abilities are agents: small Python files that give your rappter new skills.

And here's the beautiful part: one file = one skill. No config files. No dependency chains. No registration step. Drop a file in the agents folder, and your rappter learns it automatically. Like teaching a puppy to fetch by just showing it once.

The BasicAgent Pattern

Every rappter skill follows the same simple pattern: a Python class that extends BasicAgent. The class has three parts:

PartWhat It DoesThe Analogy
nameHow the rappter refers to this skillThe trick's name — "fetch," "sit," "roll over"
metadataDescription + parameters the LLM reads to decide when to use itThe card description — what's on the Pokémon card
perform()The actual code that runs when the skill is calledThe action — what happens when the puppy fetches

Your First Agent: The Joke Teller

Let's build a simple agent that tells jokes. Here's the entire file: 

import random
from basic_agent import BasicAgent

class JokeAgent(BasicAgent):
    def __init__(self):
        self.name = "JokeAgent"
        self.metadata = {
            "name": self.name,
            "description": "Tells a random joke. Call this when the user asks for a joke, wants to laugh, or needs cheering up.",
            "parameters": {
                "type": "object",
                "properties": {
                    "topic": {
                        "type": "string",
                        "description": "Optional topic for the joke (e.g. 'programming', 'animals')"
                    }
                },
                "required": []
            }
        }
        super().__init__()

    def perform(self, topic="", **kwargs):
        jokes = [
            "Why do programmers prefer dark mode? Because light attracts bugs.",
            "There are only 10 kinds of people — those who understand binary and those who don't.",
            "Why did the developer go broke? Because he used up all his cache.",
            "What's a programmer's favorite hangout place? Foo Bar.",
            "Why do Java developers wear glasses? Because they can't C#.",
        ]
        joke = random.choice(jokes)
        if topic:
            return f"Here's a joke about {topic}: {joke}"
        return joke

That's it. ~30 lines. The whole skill. Let's break down what happens:

  • name — The LLM knows this skill as "JokeAgent"
  • metadata.description — The LLM reads this to decide when to call it. This is the most important part — it's the card description.
  • metadata.parameters — What inputs the skill accepts (optional topic here)
  • perform() — The actual action. Must accept **kwargs. Returns a string.

How Auto-Discovery Works

You don't need to register agents anywhere. The brainstem automatically discovers any file matching *_agent.py in the agents directory. Here's what happens on startup:

🧠 Brainstem starts up │ ├── Scans agents/ for *_agent.py files │ ├── Imports each file │ └── Finds classes extending BasicAgent │ ├── Calls to_tool() on each agent │ └── Converts metadata to OpenAI function schema │ └── Registers tools with the LLM └── Now the LLM knows about this skill and will call it when appropriate

Drop a file in, restart, done. The rappter learned a new trick.

One File vs The World

Here's why the one-file-one-skill pattern matters. Look at what other platforms require to add a single capability:

LangChain: Tool class + chain config + prompt template

CrewAI: Agent YAML + task YAML + crew config

AutoGen: Agent class + group chat config + orchestrator

Multiple files, multiple formats, framework lock-in

🦎 RAPP Brainstem

One Python file

~30 lines

Drop in agents/, restart

No config. No YAML. No registration.

And because each skill is one self-contained file, they're tradeable like Pokémon cards:

  • Trade — Send the file to a friend. Their rappter learns it instantly.
  • Collect — Build a library of skills from the community.
  • Remix — Open it, read ~30 lines, change what it does.
  • Share — Post it anywhere. No package manager needed.

    def perform(self, **kwargs):
