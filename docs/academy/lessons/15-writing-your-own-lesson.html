Writing Your Own Lesson

The greatest skill isn't learning. It's teaching.

📖 Story

There's an old saying: if you can't explain it to a six-year-old, you don't really understand it.

Think about the best teacher you ever had. They didn't just know the material — they knew how to make you feel it. They turned abstract concepts into stories. They gave you something to try, not just something to read. They left you with an insight that changed how you saw things.

That's exactly what this academy does. Story → Try It → Insight → Next Step. And now it's your turn to write one.

This whole academy was built lesson by lesson, by people who learned something and wanted to share it. Now it's your turn to add one. What did you learn that isn't here yet?

The Lesson Format

Every lesson in RappterAcademy follows the same four-beat structure. This isn't arbitrary — it's how humans learn best:

BeatPurposeCallout Type
📖 Story Hook them emotionally. Make the concept feel like something, not just explain what it is. Use metaphors, analogies, lived experience. .callout.story
📚 Teaching The actual content. Diagrams, tables, code blocks, comparisons. Clear and concrete. No hand-waving. Regular h2 / p sections
🔬 Try It Hands-on. A command to run, a file to read, something to build. Learning by doing, not just reading. .callout.try-it
💡 Insight The "aha" moment. One paragraph that reframes everything they just learned. The thing they'll remember tomorrow. .callout.insight

After those four beats, every lesson ends with a What's Next callout (.callout.next) that bridges to the following lesson, plus a completion button.

The XP & Completion System

Each lesson has an XP value and a unique lesson ID. The completion button saves progress to localStorage and (if the academy window is open) calls back to update the skill tree. 

// In the script at the bottom of each lesson:
function completeLesson() {
  // Callback to academy skill tree
  if (window.opener && window.opener.rappterAcademy) {
    window.opener.rappterAcademy.markComplete('YOUR_ID');
  }
  // Save to localStorage
  const p = JSON.parse(
    localStorage.getItem('rappteracademy_progress') || '{}'
  );
  if (!p['YOUR_ID']) {
    p['YOUR_ID'] = {
      completed: true,
      date: new Date().toISOString().slice(0,10)
    };
  }
  localStorage.setItem(
    'rappteracademy_progress', JSON.stringify(p)
  );
  // Update button appearance
  const btn = document.getElementById('complete-btn');
  btn.textContent = '✓ Completed — YOUR_XP XP earned!';
  btn.classList.add('done');
}

Adding to the Skill Tree

To make your lesson appear in the academy, add it to the LESSONS array in index.html: 

// In docs/academy/index.html, find the LESSONS array:
{
  id: 'YOUR_ID',
  num: 16,
  title: 'Your Lesson Title',
  subtitle: 'One-line description',
  xp: 100,
  path: 'builder',     // or 'hatchling', 'apprentice'
  file: 'lessons/16-your-lesson-slug.html',
  requires: ['15']     // prerequisite lesson IDs
}

The skill tree automatically handles dependencies, progress tracking, and visual state. Your lesson just needs to be in the array.

🔬 Try It

🔬 Try It

Create your own lesson:

  1. Copy any existing lesson HTML file as your starting template
  2. Replace the content with something you learned that isn't covered yet
  3. Follow the four-beat structure: Story → Teaching → Try It → Insight
  4. Choose an XP value (50–250) based on depth and difficulty
  5. Pick a unique lesson ID and add it to the LESSONS array in index.html
  6. Test it locally — open the HTML file in your browser
  7. Submit it as a PR to the repo

Ideas for lessons that don't exist yet:

  • How to debug an agent that isn't being called
  • Writing a soul file that reflects your personality
  • Building an agent that uses an external API
  • Understanding the heartbeat daemon in depth
  • Setting up .repos.json for your team

Whatever you write, remember Article I: the story is the architecture. If a child couldn't follow your lesson, simplify it.
